# ROLES

## Назначение

Этот документ описывает роли проекта. Роль означает ответственность, а не обязательно отдельного человека.

## Management

| Роль | Главная ответственность | Основной выход |
| --- | --- | --- |
| CEO Agent | Бизнес-направление, ограничения, финальные trade-off решения | Приоритеты и go / no-go |
| CTO Agent | Техническая стратегия, архитектурные риски, технологические решения | Технический курс |
| Product Manager Agent | Ценность продукта, требования, roadmap, scope | Требования и приоритеты |
| Tech Lead Agent | Декомпозиция, техническое качество, координация разработки | Реализуемый план |

## Development

| Роль | Главная ответственность | Основной выход |
| --- | --- | --- |
| Developer Feature Agent | Реализация новой функциональности | Готовая feature-задача |
| Developer Bug Fix Agent | Диагностика и исправление багов | Исправление с проверкой |
| Developer Test Writer Agent | Тесты и regression safety | Покрытие тестами |
| Developer Code Reviewer Agent | Ревью кода и рисков | Findings и approve / request changes |

## Quality

| Роль | Главная ответственность | Основной выход |
| --- | --- | --- |
| QA Agent | Проверка пользовательских сценариев и регрессий | Test report |
| Security Agent | Риски безопасности, секреты, доступы | Security review |

## Operations

| Роль | Главная ответственность | Основной выход |
| --- | --- | --- |
| DevOps Agent | Окружения, деплой, CI/CD, наблюдаемость | Рабочий delivery path |

## Knowledge

| Роль | Главная ответственность | Основной выход |
| --- | --- | --- |
| Documentation Agent | Документация и knowledge base | Актуальные документы |

## Совмещение ролей

| Если команда маленькая | Кто может совмещать |
| --- | --- |
| CEO + Product Manager | Владелец продукта |
| CTO + Tech Lead | Технический владелец |
| Developer Feature + Bug Fix + Test Writer | Разработчик |
| QA + Documentation | Участник, закрывающий приемку |
| DevOps + Security | Технический владелец при низкой операционной сложности |

## Правило владельца

Даже если несколько людей помогают в задаче, у задачи должен быть один owner. Owner отвечает за состояние задачи, handoff и обновление документации.
