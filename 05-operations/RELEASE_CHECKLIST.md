# RELEASE_CHECKLIST

## Назначение

Этот checklist используется перед релизом, чтобы не выпускать изменения без базовой проверки продукта, качества и операций.

## Scope

- Версия:
- Дата релиза:
- Ответственный:
- Ссылка на план / задачи:

## Перед релизом

- [ ] Scope релиза понятен.
- [ ] Все задачи релиза в статусе `done` или явно исключены.
- [ ] Нет открытых P0/P1 багов без принятого решения.
- [ ] Требования обновлены.
- [ ] Changelog подготовлен.

## Проверки

- [ ] Unit tests пройдены.
- [ ] Integration / E2E tests пройдены или сознательно пропущены с причиной.
- [ ] Smoke checklist пройден.
- [ ] Security-sensitive изменения проверены.
- [ ] Миграции проверены.
- [ ] Rollback-план понятен.

## Деплой

- [ ] Production-конфигурация проверена.
- [ ] Секреты и доступы актуальны.
- [ ] Деплой выполнен.
- [ ] Post-deploy smoke пройден.
- [ ] Метрики и ошибки проверены.

## После релиза

- [ ] Команда уведомлена.
- [ ] Пользовательские / внутренние релизные заметки опубликованы.
- [ ] Новые баги заведены в [`../04-quality/BUGS.md`](../04-quality/BUGS.md) или tracker.
- [ ] Следующие действия перенесены в [`01-product/planning/PLAN.md`](../01-product/planning/PLAN.md).
