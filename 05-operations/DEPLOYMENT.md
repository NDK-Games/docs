# DEPLOYMENT

## Назначение

Этот документ описывает процесс поставки проекта в окружения: staging, production или другие целевые среды.

## Окружения

| Окружение | Назначение | URL / доступ | Кто может деплоить |
| --- | --- | --- | --- |
| development | Локальная разработка |  |  |
| staging | Проверка перед production |  |  |
| production | Пользовательская среда |  |  |

## Процесс деплоя

1. Проверить статус задач и релизный scope.
2. Запустить обязательные тесты.
3. Проверить миграции и переменные окружения.
4. Выполнить деплой.
5. Проверить smoke.
6. Обновить [`../06-history/CHANGELOG.md`](../06-history/CHANGELOG.md).
