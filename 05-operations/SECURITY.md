# SECURITY

## Назначение

Этот документ фиксирует базовую модель безопасности проекта, чувствительные данные, доступы и правила реагирования.

## Правила

- Секреты не коммитятся.
- Production-доступы выдаются по минимально необходимому принципу.
- Любой security incident получает отдельную запись и владельца.
- Новые внешние интеграции проходят проверку секретов и доступа.
- Логи не должны содержать пароли, токены и персональные данные без необходимости.

## Реакция на инцидент

1. Ограничить ущерб.
2. Отозвать или ротировать затронутые секреты.
3. Зафиксировать timeline.
4. Выпустить hotfix или rollback.
5. Проверить, что проблема не повторяется.
6. Обновить документацию и правила.
