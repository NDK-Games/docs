| `Requirement ID` | `US-*`, `FR-*` или `NFR-*` из [`REQUIREMENTS.md`](../requirements/REQUIREMENTS.md), если задача реализует продуктовое требование. |

## Definition of Ready

Перед добавлением задачи в текущий спринт:

- [ ] Цель задачи понятна.
- [ ] Есть owner.
- [ ] Есть связь с requirement, roadmap или профильным документом-владельцем.
- [ ] Понятны acceptance criteria или способ проверки.
- [ ] Известны основные зависимости и риски.

## Definition of Done

Перед закрытием текущего плана:

- [ ] Требования выполнены.
- [ ] Тесты и ручные проверки пройдены.
- [ ] Баги P0/P1 закрыты или явно приняты.
- [ ] Документация обновлена.
- [ ] Релизные / операционные шаги понятны.
