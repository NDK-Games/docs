# ADR

## Назначение

Эта папка хранит Architecture Decision Records - короткие записи о важных технических решениях.

## Когда создавать ADR

Создавайте ADR, если решение:

- трудно откатить;
- влияет на архитектуру или стоимость поддержки;
- меняет технологический стек;
- задает долгосрочный контракт;
- требует объяснения для будущих участников команды.

## Именование

```text
0001-short-title.md
0002-short-title.md
```

## Шаблон

```markdown
# ADR 0001: Title

## Статус

`proposed | accepted | superseded | rejected`

## Контекст

Почему нужно принять решение.

## Решение

Что решили.

## Альтернативы

Что рассматривали и почему не выбрали.

## Последствия

Положительные и отрицательные последствия.
```
