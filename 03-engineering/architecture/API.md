# API

## Назначение

Этот документ фиксирует внешние и внутренние API-контракты проекта: endpoints, события, команды, очереди, webhooks или другие интерфейсы.

## Общие правила

- Контракт API меняется только с обновлением этого документа.
- Breaking changes должны быть явно отмечены.
- Ошибки должны быть стабильными и пригодными для диагностики.
- Примеры запросов и ответов должны соответствовать фактической реализации.
