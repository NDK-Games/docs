# TESTING

## Назначение

Этот документ описывает стратегию тестирования проекта: что проверяем, на каком уровне и какой набор проверок обязателен перед релизом.

## Уровни тестирования

| Уровень | Что проверяет | Когда запускать | Ответственный |
| --- | --- | --- | --- |
| Unit | Чистую логику | Каждый PR | Developer |
| Integration | Связь модулей и сервисов | При изменении контрактов | Developer / QA |
| E2E / UI | Основные пользовательские сценарии | Перед релизом | QA |
| Smoke | Быструю проверку жизнеспособности | После деплоя | QA / DevOps |

## Regression checklist

- [ ] Основные сценарии работают.
- [ ] Новая функциональность покрыта тестами или ручным checklist.
- [ ] Исправленный баг не воспроизводится.
- [ ] Старые критичные сценарии не сломаны.
- [ ] Нет новых ошибок в логах.

## Критерии выхода в релиз

- Все P0/P1 баги закрыты или явно приняты ответственным.
- Пройден обязательный smoke.
- Нет неизвестных блокеров.
- Release checklist заполнен.
